# Invite-Only Onboarding

Estado: listo, pusheado a remoto y validado con usuario real.

## Hecho

- Se creó `public.tenant_invites`.
- Se agregó enum `tenant_invite_status`.
- Se implementó flujo invite-only:
  - crear invite
  - aceptar invite
  - revocar invite
- El token raw se devuelve una sola vez.
- La DB guarda solo `token_hash`.
- Owner invites quedan reservadas a `service_role`.
- Tenant admins pueden crear invites normales para su tenant.
- El callback del frontend acepta el invite y refresca sesión.

## Funciones

- `app.create_tenant_invite`
- `app.accept_tenant_invite`
- `app.revoke_tenant_invite`
- wrappers RPC públicos:
  - `public.create_tenant_invite`
  - `public.accept_tenant_invite`
  - `public.revoke_tenant_invite`
